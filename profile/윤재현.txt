기간: 2021.03 ~ 2022.02 (12개월)
역할: 풀스택 리드 개발자

프로젝트 개요:
부동산 매물 정보를 제공하고 중개사와 고객을 연결하는 플랫폼을 처음부터 개발했습니다. 지도 기반 검색, 실시간 채팅, 매물 관리 등 다양한 기능을 포함하는 복잡한 서비스였습니다.

담당 업무:

- MongoDB를 활용한 매물 데이터 관리
- Elasticsearch를 활용한 통합 검색 기능 구현

기술적 도전과 해결:

성과:
- 런칭 3개월 만에 MAU 10만 명 달성
- 지도 검색 응답 시간 평균 200ms 이하 유지
- 실시간 채팅 동시 접속자 1만 명 처리
- Google Lighthouse 성능 점수 90점 이상 달성

9. 문제해결 경험
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

서비스 런칭 직후 특정 페이지에서 간헐적으로 무한 로딩이 발생하는 문제가 있었습니다. 사용자들의 불만이 증가했고, 재현이 어려워 원인을 찾기 힘든 상황이었습니다.

문제 분석:
Sentry로 수집된 에러 로그와 사용자 세션 기록을 분석한 결과, 특정 조건에서 React useEffect가 무한 루프에 빠지는 것을 발견했습니다. 의존성 배열에 객체를 직접 넣어서 매 렌더링마다 새로운 참조가 생성되고, 이로 인해 useEffect가 계속 실행되는 문제였습니다.

해결 과정:
1. useEffect의 의존성 배열에 객체 대신 필요한 원시값만 포함하도록 수정했습니다.
2. useMemo를 활용하여 불필요한 재계산을 방지했습니다.
3. React DevTools Profiler를 활용하여 불필요한 리렌더링을 찾아 최적화했습니다.
4. ESLint의 exhaustive-deps 규칙을 엄격하게 적용하여 유사한 문제를 예방했습니다.

결과:
- 무한 로딩 문제 100% 해결
- 페이지 렌더링 성능 40% 향상
- 유사한 문제 재발 방지를 위한 코드 리뷰 체크리스트 작성
- 팀 내 React 최적화 세션 진행

이 경험을 통해 React의 렌더링 메커니즘에 대한 깊은 이해를 얻었고, 성능 최적화의 중요성을 배웠습니다.

리더십 발휘 사례:

1. 기술 스택 결정
팀원들과 함께 기술 검토 회의를 진행하여 Next.js와 NestJS를 선택했습니다. 각 기술의 장단점을 분석하고, 프로젝트 요구사항에 가장 적합한 스택을 선정했습니다. 팀원들의 학습 곡선도 고려하여 의사결정을 내렸습니다.
