【대용량 주문 처리 시스템 구축 프로젝트】
기간: 2023.04 ~ 2023.12 (9개월)
역할: 백엔드 리드 개발자

프로젝트 개요:
기존 시스템이 피크 시간대 트래픽을 감당하지 못하는 문제를 해결하기 위해 대용량 주문 처리가 가능한 새로운 시스템을 구축했습니다. 이벤트 기반 아키텍처와 비동기 처리를 도입하여 처리 성능을 대폭 향상시켰습니다.

담당 업무:

- Redis를 활용한 재고 관리 및 동시성 제어
- Spring WebFlux를 활용한 비동기 API 개발
- 성능 테스트 및 병목 지점 분석 및 개선

기술적 도전과 해결:

성과:
- 초당 처리 가능한 주문 수 500건에서 5,000건으로 10배 향상
- 평균 응답 시간 2초에서 200ms로 개선
- 시스템 가용성 99.9% 달성
- 블랙프라이데이 세일 기간 동안 무장애 운영

8. 팀워크 경험
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

이전 프로젝트에서 프론트엔드 팀과의 API 스펙 불일치로 인해 통합 단계에서 많은 수정 작업이 발생했습니다. 백엔드에서 개발한 API가 프론트엔드의 요구사항과 맞지 않아 여러 차례 수정이 필요했고, 이로 인해 일정이 지연되었습니다.

이 문제를 해결하기 위해 OpenAPI Specification을 활용한 API 문서 자동화 시스템을 도입했습니다. Swagger를 통해 API 스펙을 먼저 정의하고, 프론트엔드 팀과 리뷰한 후 개발을 시작하는 프로세스로 개선했습니다.

또한 Spring Rest Docs를 도입하여 테스트 코드에서 자동으로 API 문서가 생성되도록 했습니다. 이를 통해 실제 동작하는 API와 문서의 불일치를 방지했고, Mock Server를 제공하여 프론트엔드 팀이 백엔드 개발 완료를 기다리지 않고 병렬로 개발할 수 있도록 지원했습니다.

결과적으로 통합 단계에서의 수정 작업이 90% 감소했고, 전체 개발 기간이 30% 단축되었습니다.

9. 문제해결 경험
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

서비스 운영 중 특정 API의 응답 시간이 점점 느려지다가 결국 타임아웃이 발생하는 문제가 반복적으로 발생했습니다. 데이터베이스 커넥션 풀이 고갈되어 새로운 요청을 처리할 수 없는 심각한 상황이었습니다.

문제 분석:
APM 도구와 데이터베이스 슬로우 쿼리 로그를 분석한 결과, 특정 쿼리에서 테이블 풀 스캔이 발생하고 있었습니다. 데이터가 증가하면서 인덱스가 없는 컬럼에 대한 검색이 점점 느려지고 있었고, 이로 인해 커넥션이 오랫동안 점유되어 커넥션 풀이 고갈되었습니다.

해결 과정:
1. 슬로우 쿼리를 식별하고 적절한 인덱스를 추가했습니다.
2. N+1 쿼리 문제를 Fetch Join으로 해결했습니다.
3. 커넥션 풀 설정을 최적화하고 타임아웃 값을 조정했습니다.
4. 자주 조회되는 데이터는 Redis에 캐싱하여 DB 부하를 줄였습니다.
5. 쿼리 실행 계획을 정기적으로 모니터링하는 자동화 스크립트를 작성했습니다.

결과:
- API 평균 응답 시간 5초에서 300ms로 개선
- 데이터베이스 CPU 사용률 85%에서 40%로 감소
- 커넥션 풀 고갈 문제 완전 해결
- 시스템 안정성 대폭 향상

이 경험을 통해 데이터베이스 최적화와 성능 모니터링의 중요성을 깊이 이해하게 되었습니다.

10. 리더십 경험
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

작년에 결제 시스템 개편 프로젝트의 백엔드 파트 리더를 맡아 4명의 개발자를 이끌었습니다. 레거시 결제 시스템을 MSA 기반의 새로운 시스템으로 전환하는 중요한 프로젝트였습니다.

리더십 발휘 사례:

1. 기술 스택 결정
팀원들과 함께 기술 검토 회의를 진행하여 Spring Cloud Netflix에서 Spring Cloud Gateway로의 전환을 결정했습니다. 각 기술의 장단점을 분석하고, POC를 통해 검증한 후 의사결정을 내렸습니다.

2. 도메인 분리 및 업무 분배
결제 도메인을 주문, 결제, 정산으로 명확히 분리하고, 각 팀원의 경험과 강점을 고려하여 업무를 할당했습니다. 주니어 개발자에게는 상대적으로 복잡도가 낮은 정산 서비스를 맡기고, 시니어 개발자에게는 핵심 결제 로직을 담당하게 했습니다.

3. 코드 리뷰 문화 정착
Pull Request 기반의 코드 리뷰를 의무화하고, 리뷰 가이드라인을 작성했습니다. 단순히 문제를 지적하는 것이 아니라 더 나은 방법을 제안하고 학습할 수 있는 기회로 활용했습니다.

4. 장애 대응 프로세스 구축
서비스 런칭 전 장애 시나리오를 작성하고 대응 매뉴얼을 준비했습니다. 또한 정기적인 장애 대응 훈련을 통해 팀원들이 실제 상황에 빠르게 대처할 수 있도록 했습니다.
